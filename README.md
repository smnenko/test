# Baxity — Тестовое задание: Full Stack Developer

## О проекте

Baxity — международный онлайн-магазин цифровых товаров. В этом тестовом задании вы будете работать с упрощённой версией нашей платформы.

Проект уже содержит базовый скелет: настроенный backend (FastAPI + PostgreSQL), frontend (React + TypeScript) и Docker-окружение. Ваша задача — реализовать две фичи.

## Быстрый старт

```bash
docker compose up --build
```

- Frontend: http://localhost:5173
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs

База данных автоматически заполняется тестовыми данными при запуске.

## Задача 1: Рефакторинг каталога товаров (Frontend)

**Файл:** `frontend/src/pages/ProductsPage.tsx`

Страница каталога уже реализована и работает, но написана наспех. Ваша задача — привести код в production-ready состояние.

### Требования:
- Изучите текущий код и найдите проблемы
- Исправьте найденные проблемы
- Добавьте адаптивную вёрстку (мобильная + десктоп)
- Добавьте пагинацию

### API (уже реализован):
```
GET /api/products?category=gaming&search=steam&limit=10&offset=0
GET /api/products/{id}
```

**Оценивается:** качество кода, типизация, CSS, UX.

## Задача 2: Страница использования гифт-карты (Frontend)

**Файл:** `frontend/src/pages/RedeemPage.tsx`

Реализуйте страницу для списания баланса гифт-карты. Эта страница использует API-эндпоинт из Задачи 3.

### Требования:
- Форма ввода: код гифт-карты и сумма списания
- Отправка запроса на `POST /api/gift-cards/{code}/redeem`
- Отображение результата: обновлённый баланс карты после списания
- Обработка ошибок от API с понятными сообщениями для пользователя
- Валидация формы на клиенте

**Оценивается:** UX, обработка состояний, структура компонентов, типизация.

---

## Задача 3: Списание баланса гифт-карты (Backend)

**Файл:** `backend/app/routers/gift_cards.py`

Реализуйте API-эндпоинт для применения (списания) баланса гифт-карты.

### Требования:
```
POST /api/gift-cards/{code}/redeem
Body: { "amount": 25.00 }
```

- Найти карту по коду
- Проверить: карта активна, не просрочена, баланс достаточен
- Списать указанную сумму
- Если баланс стал 0, обновить статус на `redeemed`
- Вернуть обновлённые данные карты

### Тесты:
Напишите тесты в `backend/tests/test_gift_cards.py`.
Минимум: успешное списание, недостаточный баланс, невалидная карта.

### Тестовые данные (доступны после запуска):
Откройте http://localhost:8000/docs — там видны все эндпоинты и данные.

**Оценивается:** архитектурные решения, обработка граничных случаев, надёжность, тесты.

## Важно

- **Время:** задание рассчитано на ~2 часа
- **AI-инструменты:** можно использовать любые, это плюс. Но мы ожидаем, что вы понимаете каждую строчку своего кода.
- **Комментарии:** если вы приняли архитектурное решение, которое отличается от очевидного — оставьте короткий комментарий почему.
- **Не нужно:** Docker-конфигурации, CI/CD, деплой. Всё уже настроено, просто пишите код.
